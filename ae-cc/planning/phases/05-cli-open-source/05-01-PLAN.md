<task type="auto">
  <name>Task 1: Implement core CLI commands (daemon, sync, search, show)</name>
  <files>src/mailpilot/cli.py (rewrite)</files>
  <action>
  Rewrite the placeholder cli.py with full Click implementation:

  Global options (on the top-level group):
  - `--output` / `-o`: Literal["json", "table", "plain"], default "json"
  - `--account` / `-a`: Optional account name filter
  - `--verbose` / `-v`: Enable debug logging
  - `--config`: Path to config file (default ~/.mailpilot/config.yaml)

  Helper functions:
  - `output_result(data, format, headers=None)`: Format output as JSON (json.dumps with indent=2), table (simple column-aligned), or plain text
  - `async_command(func)`: Decorator that wraps Click commands to run async functions via `asyncio.run()`
  - `get_mailpilot(ctx) -> MailPilot`: Create/get MailPilot instance from Click context, honoring --config flag

  Daemon commands:
  - `mailpilot start` — call `run_daemon()` from daemon module (blocking)
  - `mailpilot stop` — read PID from ~/.mailpilot/daemon.pid, send SIGTERM
  - `mailpilot status` — read ~/.mailpilot/status.json, output formatted status

  Sync commands:
  - `mailpilot sync [account]` — force sync all or specific account
  - `mailpilot reindex` — rebuild Xapian index from Maildir

  Search & read commands:
  - `mailpilot search QUERY` with `--limit`, `--offset`, `--format` (alias for --output for this command)
  - `mailpilot show MESSAGE_ID` with `--raw` flag
  - `mailpilot thread THREAD_ID`
  - `mailpilot unread` with `--account` filter, `--limit`
  - `mailpilot count QUERY`

  Each command:
  1. Creates MailPilot instance
  2. Calls the appropriate API method
  3. Formats output per --output flag
  4. Outputs to stdout

  Error handling:
  - Catch MailPilot exceptions, output as JSON error: `{"error": "message", "type": "ErrorClass"}`
  - Exit code 1 on errors, 0 on success
  </action>
  <verify>mailpilot --help shows all commands. mailpilot search --help shows options.</verify>
  <done>Core CLI commands implemented: start/stop/status, sync, reindex, search, show, thread, unread, count. All output JSON by default.</done>
</task>

  - `mailpilot reply MESSAGE_ID --body TEXT [--all]`

  Management commands:
  - `mailpilot read MESSAGE_IDS...` — mark as read (variadic argument)
  - `mailpilot unread-mark MESSAGE_IDS...` — mark as unread
  - `mailpilot flag MESSAGE_IDS...`
  - `mailpilot unflag MESSAGE_IDS...`
  - `mailpilot move MESSAGE_IDS... --to FOLDER`
  - `mailpilot delete MESSAGE_IDS... [--permanent]`

  Account command group (`mailpilot account ...`):
  - `mailpilot account list` — list configured accounts
  - `mailpilot account test NAME` — test IMAP+SMTP connection, report success/failure
  - `mailpilot account add` — interactive wizard (Click prompts for each config field)
  - `mailpilot account remove NAME` — remove from config

  Tag command group (`mailpilot tag ...`):
  - `mailpilot tag add TAG MESSAGE_IDS...` — add tag to messages
  - `mailpilot tag add TAG -- QUERY` — add tag by search query (uses `--` separator)
  - `mailpilot tag remove TAG MESSAGE_IDS...`
  - `mailpilot tag list` — list all tags with counts
  - `mailpilot tag search QUERY` — shortcut for `search tag:QUERY`

  Attachment command group (`mailpilot attachment ...`):
  - `mailpilot attachment list MESSAGE_ID` — list attachments for message
  - `mailpilot attachment save MESSAGE_ID PART_ID [DIR]` — save attachment to directory

  Event command:
  - `mailpilot events [--type TYPE] [--since DURATION]` — list events

  Click group structure:
  ```
  cli (group)
  ├── start, stop, status (top-level)
  ├── sync, reindex (top-level)
  ├── search, show, thread, unread, count (top-level)
  ├── send, reply, forward (top-level)
  ├── read, unread-mark, flag, unflag, move, delete (top-level)
  ├── account (group) → list, test, add, remove
  ├── tag (group) → add, remove, list, search
  ├── attachment (group) → list, save
  └── events (top-level)
  ```

  All commands follow same pattern: parse args → create MailPilot → call API → format output → stdout.
