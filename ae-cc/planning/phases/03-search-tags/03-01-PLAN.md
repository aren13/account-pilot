  <name>Task 1: Implement Xapian indexer</name>
  <files>src/mailpilot/search/__init__.py, src/mailpilot/search/indexer.py</files>
  <action>
  Create `SearchIndexer` class:

  `__init__(self, index_path: Path, stemmer_language: str = "english")`
  - `index_path` is `~/.mailpilot/xapian/`
  - Create directory if missing
  - Xapian WritableDatabase opened lazily (single-writer pattern)
  - TermGenerator with stemmer configured

  `def index_message(self, msg: dict, body_text: str, tags: list[str])`:
  (Takes dict from db, not Pydantic model — keeps Xapian code decoupled)
  1. Create Xapian Document
  2. Set document data to `msg["mp_id"]` (used to look up in SQLite)
  3. Index with TermGenerator (stemmed):
     - Full body text
     - Subject
  4. Add boolean terms (unstemmed, exact match):
     - `XFROM:{from_address}` (lowercased)
     - `XTO:{addr}` for each recipient (lowercased)
     - `XCC:{addr}` for each CC (lowercased)
     - `XSUBJECT:{word}` for each word in subject (lowercased)
     - `XTAG:{tag}` for each tag
     - `XFOLDER:{folder}`
     - `XACCOUNT:{account_name}`
     - `XHAS:attachment` if has_attachments
     - `XMSGID:{message_id}`
  5. Add values for sorting/filtering:
     - Slot 0: date as sortable string (`xapian.sortable_serialise(timestamp)`)
     - Slot 1: account name
     - Slot 2: from_address
     - Slot 3: subject
  6. Add/replace document (use `XMSGID:{message_id}` as unique term for replace_document)
  7. Return xapian_docid

  `def remove_message(self, message_id: str)`:
  - Delete by unique term `XMSGID:{message_id}`

  `def update_tags(self, message_id: str, tags: list[str])`:
  - Get document by `XMSGID:{message_id}`
  - Remove all existing `XTAG:*` terms
  - Add new `XTAG:{tag}` terms for each tag
  - Replace document

  `def reindex(self, db: Database, get_body: Callable)`:
  - Delete entire index directory, recreate
  - Query all non-deleted messages from db
  - For each: get body from Maildir via get_body callback, index_message
  - Log progress

  `def close(self)`:
  - Close WritableDatabase (critical — corruption if not closed)
  - Flush to disk

  IMPORTANT: Only ONE writer at a time. Use a write queue or ensure single-threaded access. In asyncio context, use `asyncio.to_thread()` for all Xapian write operations since Xapian is not async.

  `search/__init__.py`: export SearchIndexer, SearchQuery
  </action>

<task type="auto">
  <name>Task 2: Implement Xapian query engine</name>
  <files>src/mailpilot/search/query.py</files>
  <action>
  Create `SearchQuery` class:

  `__init__(self, index_path: Path, stemmer_language: str = "english")`
  - Open Xapian Database (read-only)
  - Configure QueryParser with:
    - Stemmer matching index language
    - Boolean prefixes: `from:` → `XFROM:`, `to:` → `XTO:`, `cc:` → `XCC:`, `subject:` → `XSUBJECT:`, `tag:` → `XTAG:`, `folder:` → `XFOLDER:`, `account:` → `XACCOUNT:`, `has:` → `XHAS:`, `msgid:` → `XMSGID:`
    - Default operator: AND
    - Enable wildcards (FLAG_WILDCARD)
    - Enable partial matching for prefix completion
    - Spelling correction enabled (FLAG_SPELLING_CORRECTION)
    - Value range processor for date: slot 0

  `def search(self, query_string: str, limit: int = 20, offset: int = 0, sort_by: str = "relevance") -> list[dict]`:
  1. Parse query string with configured QueryParser
  2. Create Enquire object with parsed query
  3. If sort_by == "date": sort by value slot 0 (descending)
  4. If sort_by == "relevance": use default BM25 ranking
  5. Get MSet with offset and limit
  6. For each match:
     - Extract mp_id from document data
     - Get relevance score (percent)
     - Generate snippet using MSet.snippet() on the body text (or TermGenerator snippet if available)
  7. Return list of dicts: {mp_id, relevance, snippet}

  `def count(self, query_string: str) -> int`:
  - Parse query, get estimated count from MSet

  `def suggest_spelling(self, query_string: str) -> str | None`:
  - Parse query with spelling correction
  - If corrected query differs from input, return suggestion
  - Otherwise return None

  `def close(self)`:
  - Close read-only database

  Note on snippets: Xapian's built-in snippet generation (`MSet.snippet()`) requires storing the text in the document. Instead, return mp_id and let the caller fetch the preview from SQLite. The SearchResult model has a `snippet` field — populate it from the db `preview` column for now.

  Thread the read operations too: use `asyncio.to_thread()` for search calls since Xapian blocks. Create an async wrapper:
  `async def async_search(...)`, `async def async_count(...)` that wrap the sync methods.
  </action>
