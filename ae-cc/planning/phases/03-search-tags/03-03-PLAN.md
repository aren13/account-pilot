  <name>Task 1: Implement tag manager</name>
  <files>src/mailpilot/tags/__init__.py, src/mailpilot/tags/manager.py</files>
  <action>
  Create `TagManager` class:

  `__init__(self, db: Database, indexer: SearchIndexer, event_emitter: EventEmitter)`

  Tag operations (all async):
  - `async def add_tags(self, mp_ids: list[str], tags: list[str])`:
    1. For each mp_id: get message from db, get message.id
    2. For each tag: `db.get_or_create_tag(tag)` then `db.add_message_tags(message_id, tags)`
    3. Update Xapian index: `indexer.update_tags(message_id_header, all_current_tags)`
    4. Emit `email_tagged` event for each message with details: {action: "add", tags: [...]}

  - `async def remove_tags(self, mp_ids: list[str], tags: list[str])`:
    1. For each mp_id: get message, remove tags from db
    2. Update Xapian index with remaining tags
    3. Emit `email_tagged` event with details: {action: "remove", tags: [...]}

  - `async def get_tags(self, mp_id: str) -> list[str]`:
    - Get message id from db, return tag names

  - `async def list_tags(self) -> list[Tag]`:
    - Return all tags with message counts via `db.list_tags()`
    - Return as list of Tag model objects

  - `async def tag_by_query(self, action: str, tags: list[str], query: str)`:
    1. Use SearchQuery to find matching mp_ids
    2. Call add_tags or remove_tags based on action ("add"/"remove")
    - This enables: `mailpilot tag add urgent -- "from:boss and date:today"`

  Reserved tags handling:
  - Reserved tags: `inbox`, `unread`, `sent`, `draft`, `trash`, `spam`, `flagged`, `attachment`
  - These are auto-applied during sync based on IMAP flags/folder
  - Users can query them but some are managed by the system (e.g., `unread` is synced with \Seen flag)
  - Add a `RESERVED_TAGS` constant set

  `tags/__init__.py`: export TagManager, RESERVED_TAGS

<task type="auto">
  <name>Task 2: Implement auto-tag rules engine</name>
  <files>src/mailpilot/tags/rules.py</files>
  <action>
  Create `RuleEngine` class:

  `__init__(self, rules: list[RuleConfig], tag_manager: TagManager, search_query: SearchQuery, db: Database)`

  `async def evaluate_message(self, mp_id: str, message: dict)`:
  - For each rule in config:
    1. Check if message matches rule's `match` query
       - Use SearchQuery to evaluate: search the match query, check if this mp_id appears in results
       - Alternative (more efficient): parse the match query and evaluate locally against message fields for simple patterns like `from:*@domain.com`
    2. If match: apply rule actions
       - Parse each action: `"+tag"` → add tag, `"-tag"` → remove tag
       - Call tag_manager.add_tags or remove_tags
    3. Log to rule_log table: `db.execute("INSERT INTO rule_log (rule_name, message_id, actions) VALUES (?, ?, ?)")`

  `async def evaluate_batch(self, mp_ids: list[str])`:
  - For each mp_id: evaluate_message
  - Used during sync to auto-tag newly synced messages

  Simple pattern matching (for efficiency, avoid full Xapian query for simple rules):
  - `_simple_match(pattern: str, message: dict) -> bool`:
    - `from:*@domain.com` → check from_address endswith "@domain.com"
    - `from:exact@email.com` → check from_address == "exact@email.com"
    - `to:*+tag@*` → check to_addresses for plus-addressing pattern
    - `list:*` → check for List-Id or similar header
    - For complex patterns (AND, OR, NOT, date ranges): fall back to Xapian query

  Wire into sync:
  - After SyncEngine downloads new messages, call `rule_engine.evaluate_batch(new_mp_ids)`
  - Add this hook in SyncEngine's full_sync/incremental_sync methods
  </action>
