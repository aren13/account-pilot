  <action>
  Implement the JWZ threading algorithm (reference: https://www.jwz.org/doc/threading.html):

  Internal data structures:
  - `Container` class: message (Optional[dict]), parent (Optional[Container]), children (list[Container]), message_id (str)
  - `id_table: dict[str, Container]` — maps Message-ID to Container

  `class EmailThreader`:

  `def thread_messages(self, messages: list[dict]) -> list[Thread]`:
  Implementation follows JWZ steps:

  Step 1 — Build ID table:
  - For each message, create Container with message_id
  - Link parent/child via References header (each ref is parent of next)
  - In-Reply-To is the direct parent

  Step 2 — Find root set:
  - Containers with no parent form the root set
  - These are thread roots

  Step 3 — Discard empty containers:
  - If a Container has no message and exactly one child, promote the child
  - If a Container has no message and multiple children, keep as grouping node

  Step 4 — Group by subject (fallback):
  - For messages with no References/In-Reply-To, group by normalized subject
  - Normalize: strip Re:, Fwd:, [list-prefix], leading whitespace
  - Only group if subjects match exactly after normalization
  - This catches cases where mailers don't set References headers

  Step 5 — Sort:
  - Sort threads by date of most recent message (descending)
  - Sort messages within thread by date (ascending — conversation order)

  Thread ID generation:
  - `_generate_thread_id(root_message_id: str) -> str` — SHA-256 hash of root Message-ID, prefix with "t-", take first 12 hex chars. E.g., `t-a1b2c3d4e5f6`
  - Deterministic: same root message_id always produces same thread_id

  `def build_thread(self, root: Container) -> Thread`:
  - Flatten container tree into ordered message list
  - Create Thread model: thread_id, subject (from root), messages, participants (unique from/to), date (latest), message_count

  `def assign_thread_ids(self, db: Database, messages: list[dict]) -> dict[str, str]`:
  - Thread the messages
  - Return mapping of mp_id → thread_id
  - Update thread_id in database for each message

  Subject normalization:
  - `_normalize_subject(subject: str) -> str`:
    - Strip "Re: ", "RE: ", "re: " (repeating)
    - Strip "Fwd: ", "FWD: ", "fwd: " (repeating)
    - Strip "[listname] " prefix patterns
    - Strip leading/trailing whitespace
    - Return lowercased result
