<task type="auto">
  <name>Task 2: Implement config system with Pydantic models</name>
  <files>src/mailpilot/config.py, config.example.yaml</files>
  <action>
  Create Pydantic v2 models for the full config.yaml structure:

  Models (all with `model_config = ConfigDict(extra="forbid")`):
  - `AuthConfig`: method (Literal["password", "oauth2"]), password_cmd (Optional[str]), client_id/client_secret (Optional, for OAuth2)
  - `ImapConfig`: host, port (default 993), encryption (Literal["tls", "starttls", "none"]), auth (AuthConfig)
  - `SmtpConfig`: host, port (default 587), encryption (Literal["tls", "starttls", "none"]), auth (AuthConfig)
  - `FolderConfig`: watch (list[str] default ["INBOX"]), sync (list[str] default ["INBOX"]), aliases (dict[str, str] default {})
  - `AccountConfig`: name, email, display_name (Optional), provider (Literal["gmail", "outlook", "custom"] default "custom"), imap, smtp, folders (FolderConfig), webhook_url (Optional)
  - `SearchConfig`: stemming (default "english"), spelling (default True), snippet_length (default 200), default_limit (default 20)
  - `SyncConfig`: idle_timeout (default 1680), reconnect_base_delay (default 5), reconnect_max_delay (default 300), full_sync_interval (default 3600), max_message_size (default 52428800)
  - `RuleAction`: tag (str) ‚Äî the "+tag" or "-tag" format
  - `RuleConfig`: name, match (str ‚Äî Xapian query), actions (list[RuleAction])
  - `MailPilotGlobalConfig`: data_dir (default "~/.mailpilot"), log_level (default "INFO"), log_format (default "json"), log_file (Optional)
  - `MailPilotConfig`: mailpilot (MailPilotGlobalConfig), accounts (list[AccountConfig]), search (SearchConfig), sync (SyncConfig), rules (list[RuleConfig] default [])

  Config loader function `load_config(path: Path | None = None) -> MailPilotConfig`:
  - Default path: `~/.mailpilot/config.yaml`
  - Read YAML, validate with Pydantic, return typed config
  - Expand `~` in all paths (data_dir, log_file)

  Password resolution function `resolve_password(auth: AuthConfig) -> str`:
  - If method is "password" and password_cmd is set, run the command via `subprocess.run(shell=True, capture_output=True)` and return stripped stdout
  - Raise `ConfigError` if command fails or returns empty
  - NEVER store resolved passwords on the config object ‚Äî resolve on demand only

  Create `config.example.yaml` with the full example from the PRD (Section 6), using placeholder values.
  </action>
  <verify>Write a test in tests/test_config.py: load config.example.yaml (with password_cmd replaced by a mock like "echo test123"), validate all fields parse correctly, test default values, test validation errors for missing required fields.</verify>
  <done>Config loads from YAML, validates all fields, resolves password_cmd, raises clear errors on invalid config. config.example.yaml matches the PRD spec exactly. Tests pass.</done>
