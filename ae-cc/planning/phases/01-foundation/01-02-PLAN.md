<task type="auto">
  <name>Task 2: Implement SQLite database layer with migrations</name>
  <files>src/mailpilot/database.py</files>
  <action>
  Create async database layer using aiosqlite:

  `Database` class:
  - `__init__(self, db_path: Path)` — store path, connection is None until init
  - `async def initialize(self)` — create connection, enable WAL mode, enable foreign keys, run migrations
  - `async def close(self)` — close connection gracefully
  - Context manager support (`__aenter__`, `__aexit__`)

  Migration system:
  - Create `schema_version` table: `version INTEGER, applied_at TIMESTAMP`
  - `MIGRATIONS` list of (version_number, sql_string) tuples
  - Migration 1: Full schema from PRD Section 9 (accounts, messages, tags, message_tags, rule_log, outbox, events — all tables + all indexes)
  - `async def _run_migrations(self)` — check current version, apply pending migrations in order

  mp_id generation:
  - `async def _next_mp_id(self) -> str` — query max id from messages, return `f"mp-{next_id:06d}"`

  Query helpers (all async):
  - `insert_account(name, email, display_name, provider) -> int`
  - `get_account(name) -> dict | None`
  - `get_account_by_id(id) -> dict | None`
  - `list_accounts() -> list[dict]`
  - `insert_message(**kwargs) -> int` — auto-generates mp_id
  - `get_message(mp_id) -> dict | None`
  - `get_message_by_message_id(message_id) -> dict | None`
  - `get_messages_by_thread(thread_id) -> list[dict]`
  - `update_message(mp_id, **kwargs)` — updates specified fields + updated_at
  - `delete_message(mp_id)` — sets is_deleted=1
  - `search_messages(account_id=None, folder=None, is_deleted=False, limit=20, offset=0) -> list[dict]`
  - `insert_tag(name) -> int`
  - `get_or_create_tag(name) -> int`
  - `add_message_tags(message_id: int, tag_names: list[str])`
  - `remove_message_tags(message_id: int, tag_names: list[str])`
  - `get_message_tags(message_id: int) -> list[str]`
  - `list_tags() -> list[dict]` — with message counts
  - `insert_event(account_id, event_type, message_id=None, details=None) -> int`
  - `get_events(event_type=None, since=None, limit=50) -> list[dict]`
  - `insert_outbox(**kwargs) -> int`
  - `update_outbox(id, **kwargs)`
  - `get_pending_outbox() -> list[dict]`

  Use parameterized queries everywhere (never f-strings in SQL). Use `aiosqlite.Row` with `row_factory = aiosqlite.Row` for dict-like access.
  </action>
