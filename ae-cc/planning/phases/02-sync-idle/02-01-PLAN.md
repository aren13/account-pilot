  <name>Task 1: Implement Maildir storage layer</name>
  <files>src/mailpilot/imap/sync.py</files>
  <action>
  Create `MaildirManager` class for local Maildir operations:

  `__init__(self, base_path: Path)` — base_path is `~/.mailpilot/maildir/`

  Maildir operations:
  - `ensure_maildir(account: str, folder: str) -> Path` — create `base_path/account/folder/{cur,new,tmp}/` directories if missing, return folder path
  - `save_message(account: str, folder: str, uid: int, raw_bytes: bytes, flags: list[str]) -> Path` — write to tmp/ first (atomic), then move to new/ or cur/ based on flags. Filename format: `{timestamp}.{uid}.{hostname}:2,{flags}` (standard Maildir naming). Return final path.
  - `get_message_path(account: str, folder: str, uid: int) -> Path | None` — find message file by UID in cur/ or new/
  - `read_message(path: Path) -> bytes` — read raw bytes from Maildir file
  - `update_flags(path: Path, flags: list[str]) -> Path` — rename file to update flag suffix in Maildir filename, return new path. Flag chars: S=Seen, F=Flagged, R=Replied, D=Draft, T=Trashed.
  - `delete_message(path: Path)` — remove file from disk
  - `list_uids(account: str, folder: str) -> set[int]` — scan cur/ + new/, extract UIDs from filenames

  Flag mapping (Maildir char ↔ IMAP flag):
  - S ↔ \Seen
  - F ↔ \Flagged
  - R ↔ \Answered
  - D ↔ \Draft
  - T ↔ \Deleted

  Important: All file operations must be atomic where possible (write to tmp/ then rename). Never leave partial files in cur/ or new/.
  </action>

  <done>MaildirManager creates directories, saves/reads/deletes messages atomically, handles flag encoding in filenames correctly.</done>

<task type="auto">
  <name>Task 2: Implement sync engine (full + incremental)</name>
  <files>src/mailpilot/imap/sync.py (extend)</files>
  <action>
  Create `SyncEngine` class in the same file, below MaildirManager:

  `__init__(self, imap_client: ImapClient, db: Database, maildir: MaildirManager, account: AccountConfig)`

  Sync methods:
  - `async def full_sync(self, folder: str, progress_callback: Callable | None = None)`:
    1. Fetch all UIDs from server via `imap_client.fetch_uids(folder)`
    2. Get local UIDs from `maildir.list_uids(account, folder)` + db query
    3. New on server (server_uids - local_uids): download each → maildir.save_message → parse → db.insert_message → fire progress callback
    4. Deleted on server (local_uids - server_uids): mark is_deleted in db, optionally remove from maildir
    5. Flag sync for existing messages: fetch server flags, compare with local, update whichever is newer (server wins on conflict)
    6. Report progress via callback: `(downloaded, total, current_subject)`

  - `async def incremental_sync(self, folder: str, last_uid: int) -> list[str]`:
    1. Fetch UIDs > last_uid from server
    2. Download each new message → maildir → parse → db → return list of new mp_ids
    3. Much faster than full_sync — only fetches truly new messages

  - `async def sync_flags(self, folder: str)`:
    1. For all non-deleted messages in folder: fetch server flags
    2. Compare with db flags (stored as JSON list)
    3. Server wins: update local db + maildir filename
    4. Also push local flag changes to server (for mark_read, flag, etc. done while offline)

  - `async def sync_account(self, progress_callback: Callable | None = None)`:
    1. For each folder in account.folders.sync: full_sync or incremental_sync based on whether we've synced before (check db for last_sync)
    2. Update account.last_sync in db

  Message deduplication:
  - Before inserting: check if message_id already exists in db for this account
  - If duplicate: skip (don't re-download)
  - Log duplicates for debugging

  Error handling:
  - Individual message download failures: log error, skip, continue with rest
  - Connection drops mid-sync: log, return partial results (caller can retry)
  - Don't let one bad message crash the entire sync
  </action>

  <done>Full sync downloads all messages, incremental sync only downloads new ones, flag sync is bidirectional with server-wins, deduplication by Message-ID works.</done>
