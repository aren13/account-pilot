<task type="auto">
  <name>Task 1: Implement IMAP IDLE listener</name>
  <files>src/mailpilot/imap/idle.py</files>
  <action>
  Create `IdleListener` class — one asyncio task per (account, folder):

  `__init__(self, imap_client: ImapClient, sync_engine: SyncEngine, account: AccountConfig, folder: str, config: SyncConfig, on_new_mail: Callable | None = None)`

  Core loop (`async def run(self)`):
  1. Connect to IMAP, SELECT folder
  2. Get initial highest UID from server
  3. Enter IDLE: `await imap_client.idle_start(folder)` (aioimaplib's idle method)
  4. Wait for notification OR timeout (idle_timeout from config, default 1680s — under 29-min RFC limit)
  5. On notification:
     a. Break IDLE: `await imap_client.idle_done(folder)`
     b. Fetch new UIDs since last known UID
     c. Trigger incremental_sync for new messages
     d. Call `on_new_mail` callback with list of new mp_ids
     e. Resume IDLE (go to step 3)
  6. On timeout (no notification): break IDLE, immediately re-IDLE (step 3) — this prevents server-side timeout
  7. On connection error: log, wait (exponential backoff), reconnect, resume from step 1

  Important IDLE gotchas to handle:
  - Re-IDLE BEFORE the 29-minute RFC timeout — use idle_timeout (1680s = 28 min)
  - aioimaplib IDLE returns an `asyncio.Future` — await it with timeout
  - Some servers send spurious "OK Still here" responses during IDLE — filter these, only act on EXISTS notifications
  - Gmail IDLE sometimes misses messages sent via API — this is expected, periodic sync handles it

  `async def stop(self)`:
  - Set a `_running = False` flag
  - If currently in IDLE, break it
  - Close the connection

  `@property def is_running(self) -> bool`
  </action>

  <done>IDLE listener runs continuously, detects new mail, triggers sync, handles timeouts and reconnection, stops cleanly.</done>

<task type="auto">
  <name>Task 2: Implement daemon orchestrator</name>
  <files>src/mailpilot/daemon.py</files>
  <action>
  Create `MailPilotDaemon` class — the main long-running process:

  `__init__(self, config: MailPilotConfig)`

  `async def start(self)`:
  1. Initialize database
  2. For each account in config:
     a. Create ImapClient
     b. Create SyncEngine (with MaildirManager)
     c. Run initial full sync for all configured folders
     d. Register account in db if not exists
  3. For each (account, folder) in watch lists:
     a. Create IdleListener
     b. Launch as asyncio task
  4. Launch periodic full-sync safety net: asyncio task that runs full_sync every `full_sync_interval` seconds (default 3600 = 1 hour) — catches anything IDLE missed
  5. Log: "MailPilot started: {N} accounts, {M} IDLE listeners"

  `async def stop(self)`:
  1. Log: "Shutting down..."
  2. Cancel periodic sync task
  3. Stop all IDLE listeners (await each stop())
  4. Disconnect all IMAP clients
  5. Close database
  6. Log: "MailPilot stopped"

  Signal handling (in the module-level `run_daemon()` function):
  - Register SIGTERM and SIGINT handlers via `asyncio.get_event_loop().add_signal_handler()`
  - On signal: call `daemon.stop()`, then exit cleanly

  Status reporting:
  - `async def status(self) -> dict` — returns: accounts (list of AccountStatus), idle_listeners (count, which folders), last_sync times, uptime
  - Write status to `~/.mailpilot/daemon.pid` (PID file) on start, remove on stop
  - Write status JSON to `~/.mailpilot/status.json` periodically (every 60s)

  `run_daemon(config: MailPilotConfig)` — module-level entry point:
  - Create daemon, register signals, run `daemon.start()`, wait until stopped
  </action>
